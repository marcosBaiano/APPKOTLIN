4. ENDPOINTS DA API
================================================================================

BASE: http://localhost:8080/api

COLETAS (com autenticação JWT onde indicado):
├── GET    /coletas                           → Listar todas
├── GET    /coletas/{id}                      → Detalhes
├── GET    /coletas/status/{status}           → Por status
├── GET    /coletas/stats                     → Estatísticas
├── GET    /coletas/user                      [AUTH] → Do usuário
├── POST   /coletas                           [AUTH] → Criar
├── PUT    /coletas/{id}/status               [AUTH] → Atualizar status
└── DELETE /coletas/{id}                      [AUTH] → Deletar

DEPOIMENTOS:
├── GET    /depoimentos                       → Listar todos
├── GET    /depoimentos/{id}                  → Detalhes
├── GET    /depoimentos/user                  [AUTH] → Do usuário
├── POST   /depoimentos                       [AUTH] → Criar
└── DELETE /depoimentos/{id}                  [AUTH] → Deletar

6. INSTRUÇÕES DE EXECUÇÃO
================================================================================

PASSO 1: Garantir que MySQL está rodando
  $ mysql -u root -p edneiaSQL
  > CREATE DATABASE IF NOT EXISTS appkotlin;

PASSO 2: Instalar Java (se não tiver)
  - Download: https://www.oracle.com/java/technologies/downloads/
  - Adicionar ao PATH

PASSO 3: Compilar o projeto
  $ cd c:\www\APPKOTLIN
  $ .\gradlew build

PASSO 4: Executar o servidor
  $ .\gradlew run
  
  OU
  
  $ java -jar build/libs/APPKOTLIN-1.0-SNAPSHOT.jar

PASSO 5: Acessar o frontend
  - Abrir navegador: http://localhost:8080
  - Fazer login com credenciais

9. FLUXO DE DADOS
================================================================================

CLIENTE:
1. Acessa http://localhost:8080
2. Faz login (credenciais salvas localmente e JWT obtido)
3. Vê dashboard com estatísticas
4. Cria nova solicitação de coleta
5. Upload de foto em base64
6. Solicitação enviada via POST /api/coletas (com token JWT)
7. Recebe ID da coleta criada
8. Acompanha status em tempo real

ADMIN:
1. Faz login como ADMIN
2. Vê fila geral de coletas (GET /api/coletas)
3. Pode atualizar status de coletas (PUT /api/coletas/{id}/status)
4. Transitando: Pendente → Em Trânsito → Finalizada
5. Vê rotas e despachos

DEPOIMENTOS:
1. Usuário cria depoimento (POST /api/depoimentos)
2. Armazenado no banco com userId
3. Listado em tempo real para todos (GET /api/depoimentos)
4. Pode ser deletado pelo autor

10. PROBLEMAS CONHECIDOS E SOLUÇÕES
================================================================================

❌ JAVA_HOME não configurado
   ✅ Solução: Instalar Java e adicionar ao PATH do Windows

❌ Porta 8080 já em uso
   ✅ Solução: Mudar em Main.kt ou encerrar processo
   
❌ Banco de dados não conecta
   ✅ Solução: Verificar MySQL rodando, credenciais em DatabaseConfig.kt

❌ CORS error no frontend
   ✅ Solução: Verificar se CORS plugin está instalado ou usar URL correta

❌ Token JWT expirado
   ✅ Solução: Fazer login novamente (token renovado)
