FLUXOS DE DADOS IMPLEMENTADOS
================================================================================

FLUXO DE COLETA (Cliente):
1. Usuario faz login → POST /users/login
2. Recebe JWT token → localStorage
3. Acessa dashboard → GET /coletas/stats
4. Cria coleta com foto → POST /coletas (com JWT)
5. Recebe ID da coleta
6. Acompanha status → GET /coletas/user (com JWT)
7. Admin atualiza → PUT /coletas/{id}/status (com JWT)
8. Cliente vê atualização

FLUXO DE DEPOIMENTO:
1. Usuario autenticado acessa mural
2. Cria depoimento → POST /depoimentos (com JWT)
3. Depoimento armazenado no BD
4. Todos veem novo depoimento → GET /depoimentos
5. Usuário pode deletar seu depoimento → DELETE /depoimentos/{id}

FLUXO DE ADMINISTRAÇÃO:
1. Admin faz login como ADMIN
2. Vê fila geral → GET /coletas
3. Clica em "DESPACHAR" → PUT /coletas/{id}/status = "Em Trânsito"
4. Clica em "CONFIRMAR" → PUT /coletas/{id}/status = "Finalizada"
5. Sistema registra timestamp automaticamente
